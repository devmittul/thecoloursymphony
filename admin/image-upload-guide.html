Image Upload Guide

This guide explains how images work with your GitHub-based art website. Understanding this will help you properly manage your artwork images.

How Images Flow in Your System

1. Upload in Admin Panel
2. Save to /images folder
3. Push to GitHub
4. Display on Website

Why GitHub Doesn't Store Your Images Automatically

When you upload an image through the admin panel, the image data is only stored temporarily in your browser. Our system is designed this way because:

  • GitHub is primarily for code, not large media files
  • Automatic image uploads would require server-side code
  • This approach gives you more control over your image assets

Important to Understand:

Only the metadata about your artwork (title, description, image path, etc.) is stored in the GitHub JSON database. The actual image files must be manually placed in your images folder and pushed to GitHub.

Step-by-Step Process

1

Upload Image in Admin Panel

When you upload an image in the "Add Artwork" form, the system processes it and shows you a preview.

The system will create a path for the image like: images/your_filename.jpg

2

Save the Image File to Your Computer

If you're using drag and drop, your original image is still on your computer. You need to:

  1. Create a copy of that image
  2. Name it exactly as shown in the admin panel
  3. Place it in the /images folder of your project

Example: If your artwork form shows:

images/sunset_painting.jpg

You must save your image as "sunset_painting.jpg" in the images folder.

3

Export the Artwork Data

On the "Manage Artworks" page, click the "Export Data for GitHub" button.

This will generate a JSON file that contains all of your artwork information.

4

Upload Both to GitHub

You need to upload both to your GitHub repository:

  1. The artworks.json file (to the data folder)
  2. Your image files (to the images folder)

You can do this through the GitHub website or using Git commands.

Tips for Success

  • Use consistent file names - Don't change the image filename after uploading
  • Optimize your images - Compress images before uploading to keep your site fast
  • Create backups - Keep copies of your artwork images separately
  • Check paths carefully - Ensure the paths in your JSON match your actual image files

Advanced: Using Other Image Hosting Options

If you'd prefer not to store images on GitHub, you can use external image hosting services:

  • Cloudinary - Offers generous free tier for image hosting
  • Imgur - Free image hosting with direct links
  • Firebase Storage - Google's cloud storage solution

With these services, just paste the full URL into the image path field instead of a relative path.
